# 自我检查与润色（写完正文后的最后一轮）

目标：把文章从“写完了”提升到“读者真的看懂了”，并确保满足最初设定的博客要求（语气自然、逻辑严谨、不瞎猜、少列表、多自然段、图文对应、引用可核验等）。

建议把这一轮当成“发布前 QA”，按顺序走完。

## 0) 先跑硬性检查（自动化）

推荐顺序：
1. 结构/格式/禁忌项（标题、元信息、参考文献、整页图禁用等）先查一遍（不强制 URL）：
   - `python scripts/check_article_requirements.py <blog.md>`
2. 图片 URL 回填（若正文里仍是本地图片路径）：
   - `python scripts/sync_lsky_images.py --workspace <项目目录> --markdown <blog.md>`
3. 再跑一次结构检查（要求全部 https URL）：
   - `python scripts/check_article_requirements.py <blog.md> --require-urls`
4. 字数范围（目标 5000-7000）：
   - `python scripts/check_article_length.py <blog.md> --min 5000 --max 7000`

原则：自动检查不过，就先改到通过，再进入下一步。

## 1) “讲明白了吗？”——读者复述测试（强烈推荐）

把自己当成读者（或请同事/朋友代读 TL;DR），看能否在 60 秒内回答下面 5 个问题：

1. 这篇论文在解决什么任务？输入/输出/约束是什么？
2. 以前方法主要卡在哪？这篇工作要解决的核心矛盾是什么？
3. 它的关键设计点是什么？为什么这样设计能解决矛盾？
4. 它最硬的实验结论是哪 2-3 条？证据来自哪张图/哪个表？
5. 我如果要复现，最短闭环要做什么？关键开关有哪些？

如果任何一题回答含糊：
- 回到正文对应段落补充“具体化信息”（输入/输出/阈值/指标定义/数据设置/最短命令）。
- 给出“最小必要背景”：只解释读者必须知道的领域黑话，不要写成科普长文。

## 2) “讲清楚了吗？”——证据链与边界检查

逐条检查文章中“看起来像结论”的句子，确保满足：

- 每条结论都能指出证据来源：论文哪一节/哪张图（或代码里哪段实现思路）。
- 不出现主观臆断：作者动机、未来展望、没在输入里出现的数字/对比。
- 如果输入不足以确认：明确写“无法从当前输入确认/论文未说明”。

高风险点（重点扫）：
- 绝对化措辞：一定/必然/显著/完全优于……
- 模糊化措辞：很强/很大/明显……但没给证据。

## 3) 术语与比喻：够“接地气”但不误导

- 每个领域术语首次出现时：用一句中文解释它在这里是什么意思。
- 比喻可以用，但必须能回到精确定义；不允许用比喻替代关键机制。
- 把“论文句子”翻译成“工程读法”：这一步对应输入什么、输出什么、在哪一步起作用。

## 4) 结构与可读性：像文章，不像 PPT

检查以下约束是否满足（越靠上越重要）：
- 自然段为主，列表仅用于总结要点；避免一屏全是 bullet。
- 章节标题不要过碎；小标题太多就合并成自然段过渡。
- 段落长度控制在 3-7 句左右，避免超长段落堆叠。
- 段与段之间有“承上启下”的一句话，不要硬切。

## 5) 图文一致性（必须）

- 插图只能来自 `paper/figures/` 与 `paper/front_matter/` 的裁剪图，禁止整页 `paper/pages/`。
- 每张图都要在正文里“解释它说明了什么”，不能只贴图不解读。
- 图注用中文写清楚这张图在支撑哪条结论；不要直接复制英文 caption 一长段。
- 完稿后应全部是 https URL（Lsky），不要残留本地路径。

## 6) 元信息与期刊指标（可核验）

- 影响因子/分区/Top：来自本地库查询结果；查不到就写未收录。
- DOI 必须是输入中出现的 DOI（不要猜）。

## 7) 参考文献（不造假）

- 必须有“参考文献”小节。
- `[1]` 必须是本次论文；信息仅来自输入文本（不要补作者、补年份）。
- 其他引用只收录输入中明确出现的 DOI/arXiv/URL。

## 8) 最后一遍润色（只做减法）

最后一遍不要再加新信息，只做“表达层面”的修整：
- 删重复：同一个观点出现两遍就合并。
- 改抽象：把“更好/更强”改成“在哪个指标上更好、好多少、证据在哪”。
- 改语气：去掉夸张营销词（如“颠覆/震撼/史诗级”），保持朴素但有力量。
- 改连贯：每一节开头补一句“这一节要解决的问题”，结尾补一句“这一节得到的结论”。

做到以上 8 步，基本就能稳定满足“讲明白、讲清楚”以及最初博客写作要求。
